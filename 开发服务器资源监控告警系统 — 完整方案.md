# srvpulse — 完整方案

> 版本：1.2.0 | 目标环境：Linux（生产）/ Windows（开发调试） | 部署方式：Git + 单机轻量部署

## 一、方案概述

### 1.1 背景与目标

开发一个运行在开发环境服务器上的轻量级监控脚本，定时采集系统资源使用率，当 CPU、内存或磁盘使用率**连续多次**超过阈值时，通过**飞书自定义机器人**（带签名校验）发送告警消息。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **轻量** | 单 Python 脚本 + 配置文件 + Cron，无需额外服务 |
| **可靠** | 状态持久化，事务级文件锁，发送成功才标记已告警 |
| **安全** | 飞书签名校验，配置文件 600 权限，状态文件 600 权限 |
| **可运维** | Git 部署 + 一键脚本，日志可查，内置测试模式 |
| **跨平台** | Linux 生产部署，Windows 本地开发调试 |

### 1.3 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.6 - 3.13+ | 兼容老服务器，pip 自动解析依赖版本 |
| 系统指标 | `psutil` | 成熟稳定，跨平台 |
| HTTP 客户端 | `requests` | 标准库，简洁 |
| 配置管理 | `PyYAML` | 可读性好，易维护 |
| 定时调度 | Cron | 系统级可靠，无需常驻 |
| 状态存储 | JSON 文件 | 轻量无依赖 |
| 签名算法 | `hmac` + `hashlib` | Python 内置，无额外依赖 |
| 文件锁 | `fcntl.flock` / `msvcrt.locking` | 按平台自动适配 |
| 代码分发 | Git | 用户执行 `git clone` / `git pull` |
| 安装配置 | `srvpulse` | install / uninstall / start / stop / status |

## 二、核心功能设计

### 2.1 资源采集

| 指标 | 采集方式 | 说明 |
|------|----------|------|
| CPU 使用率 | `psutil.cpu_percent(interval=1)` | 采样 1 秒平均值 |
| 内存使用率 | `psutil.virtual_memory().percent` | 系统整体内存占用 |
| 磁盘使用率 | `psutil.disk_usage(path).percent` | 支持自动发现 + 手动配置 |

### 2.2 磁盘自动发现

生产环境常见 Docker/K8s 服务器，`df` 中大量 **overlay** 挂载显示 **100%**，这是容器层文件系统特性，不代表真实硬盘已满，**不应告警**。

**策略：**

1. `auto_discover: true` 时，通过 `psutil.disk_partitions()` 自动发现真实磁盘挂载
2. 默认排除虚拟/容器文件系统：`overlay`、`aufs`、`tmpfs`、`devtmpfs`、`proc`、`sysfs` 等
3. 默认排除 Docker 相关路径：`/var/lib/docker`、`/overlay2/`、`/merged` 等
4. 按物理设备去重，同一硬盘多个挂载点只监控一次
5. 手动 `paths` 可补充或覆盖特定挂载点的阈值

```
psutil.disk_partitions()
    → 过滤 exclude_fstypes（overlay/tmpfs/...）
    → 过滤 exclude_path_contains（/docker//overlay2/...）
    → 按 device 去重
    → 逐个采集使用率，独立告警
```

**每个挂载点独立判断：**

- 使用率 > 该挂载点 threshold → 连续计数 +1
- 连续达标且未告警 → 发送飞书消息
- 恢复正常 → 计数归零，清除告警标记
- **不需要**所有磁盘都满才告警，也**不会**因 Docker overlay 100% 误报

### 2.3 告警判定规则

- 每个指标**独立维护**连续超标计数和告警状态
- **只有发送成功才标记已告警**，避免网络故障导致告警丢失
- 判定逻辑：

```
指标当前值 > 阈值
    → 连续计数 +1
    → 计数 >= 连续次数(默认3次) 且 未告警
        → 尝试发送告警
        → 发送成功 → 标记已告警
        → 发送失败 → 不标记（下次继续尝试）
指标当前值 <= 阈值
    → 连续计数归零，告警标记清除
```

### 2.4 飞书告警集成

根据飞书官方文档，签名算法实现如下：

```python
def gen_feishu_sign(timestamp: int, secret: str) -> str:
    string_to_sign = f"{timestamp}\n{secret}"
    hmac_code = hmac.new(
        string_to_sign.encode("utf-8"),
        msg=b"",
        digestmod=hashlib.sha256
    ).digest()
    return base64.b64encode(hmac_code).decode("utf-8")
```

### 2.5 跨平台文件锁

| 平台 | 实现 | 注意点 |
|------|------|--------|
| Linux | `fcntl.flock(LOCK_EX)` | 锁整个文件 |
| Windows | `msvcrt.locking` | 加锁/解锁前必须 `seek(0)`，否则 PermissionError |

状态文件采用**原地写入**（truncate + write + fsync），避免 rename 导致锁失效。

### 2.6 错误处理策略

| 异常场景 | 处理方式 |
|----------|----------|
| Python 版本 < 3.6 | 启动时明确报错退出 |
| 配置文件不存在/格式错误 | 打印错误退出（exit 1），不执行采集 |
| 配置项校验失败 | 明确提示缺失项，exit 1 |
| psutil 采集失败 | 记录错误日志，跳过该指标 |
| 状态文件读写失败 | 使用默认状态继续执行 |
| 状态文件损坏 | 自动重置，记录警告日志 |
| 飞书 HTTP 错误（4xx/5xx） | 记录状态码，不标记已告警 |
| 飞书返回非 JSON | 记录错误，不标记已告警 |
| 飞书返回业务错误 | 记录错误码和消息，不标记已告警 |
| 网络超时/不可达 | 重试 3 次（间隔 1 秒），失败则不标记已告警 |
| Windows 控制台编码 | `--test` 输出使用 `[OK]`/`[FAIL]`，避免 GBK emoji 报错 |

## 三、配置设计

### 3.1 配置文件 `config.yaml`

```yaml
# ========== 基础配置 ==========
# 日志级别: DEBUG, INFO, WARNING, ERROR
log_level: INFO

# 连续超标次数（达到此次数后触发告警）
consecutive_count: 3

# ========== 监控项配置 ==========
monitors:
  # CPU监控
  cpu:
    enabled: true
    threshold: 90              # 告警阈值（百分比）

  # 内存监控
  memory:
    enabled: true
    threshold: 85

  # 磁盘监控（支持自动发现真实硬盘，排除 Docker overlay 等虚挂载）
  disk:
    enabled: true
    auto_discover: true         # 自动发现所有真实磁盘挂载点
    default_threshold: 90        # 自动发现挂载点的默认告警阈值
    paths: []                    # 可选：手动补充或单独指定阈值
      # - path: /data
      #   threshold: 85
    # exclude_fstypes:           # 可选：额外排除的文件系统类型
    #   - overlay
    # exclude_path_contains:     # 可选：路径包含以下片段则跳过
    #   - /docker/

# ========== 飞书机器人配置（必填） ==========
feishu:
  webhook_url: "https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxxxx"
  secret: "YOUR_SECRET_KEY"
  timeout: 5                    # 请求超时时间（秒），必须 > 0
  max_retries: 3                # 失败重试次数，必须 >= 0
```

**磁盘配置说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `auto_discover` | bool | 是否自动发现真实磁盘挂载，默认 `false`（校验时未配置则 false） |
| `default_threshold` | int | 自动发现挂载点的默认阈值，默认 90 |
| `paths` | list | 手动补充挂载点，可单独设阈值；与自动发现合并 |
| `exclude_fstypes` | list | 额外排除的文件系统类型，默认内置 overlay/tmpfs 等 |
| `exclude_path_contains` | list | 路径包含关键字则跳过，默认内置 docker 相关路径 |

> 校验规则：`auto_discover` 和 `paths` 至少启用其一。

### 3.2 状态文件 `state.json`

```json
{
    "cpu": {
        "count": 2,
        "alerted": false
    },
    "memory": {
        "count": 0,
        "alerted": false
    },
    "disk_/": {
        "count": 1,
        "alerted": false
    },
    "disk_/data": {
        "count": 0,
        "alerted": false
    }
}
```

每个磁盘挂载点使用独立 key（`disk_{挂载路径}`），互不影响。

## 五、部署方案

### 5.1 部署方式（Git）

**推荐流程：**

```bash
# 服务器上执行
sudo git clone git@github.com:gejigang2008/srvpulse.git /opt/srvpulse
cd /opt/srvpulse
sudo ./srvpulse install              # 交互式配置飞书 → 校验 → 安装 Cron
```

强制交互配置：`sudo ./srvpulse install --interactive`

非交互环境需事先编辑好 `config.yaml` 再执行 `srvpulse install`。

**更新版本：**

```bash
cd /opt/srvpulse
git pull
sudo ./srvpulse install
```

### 5.2 安装路径

| 项目 | 路径 |
|------|------|
| 安装目录 | `/opt/srvpulse/` |
| 日志文件 | `/var/log/srvpulse.log` |
| Cron 配置 | `/etc/cron.d/srvpulse` |
| 日志轮转 | `/etc/logrotate.d/srvpulse` |

### 5.3 CLI 命令 `srvpulse`

| 子命令 | 说明 |
|--------|------|
| `install` | 安装/更新：依赖、飞书配置、Cron、logrotate，并链接到 `/usr/local/bin/srvpulse` |
| `uninstall` | 完全卸载 |
| `start` | 启用 Cron 定时监控 |
| `stop` | 禁用 Cron（保留备份） |
| `status` | 查看安装目录、运行状态、飞书配置、最近日志、配置检查 |

### 5.4 卸载

使用 `sudo srvpulse uninstall`。

### 7.6 常见问题排查

| 问题现象 | 排查方法 |
|----------|----------|
| 脚本不执行 | `grep CRON /var/log/syslog \| grep srvpulse` |
| 配置文件错误 | `python monitor.py --check` |
| 飞书收不到消息 | `python monitor.py --test` |
| 飞书签名失败 | 检查 `feishu.secret` 是否与飞书控制台一致 |
| Docker overlay 误报 | 确认 `auto_discover: true`，运行 `--check` 查看目标列表 |
| 漏监控某个盘 | `--check` 查看目标，必要时在 `paths` 手动补充 |
| Git 拉取失败 | 检查服务器 SSH 密钥或改用 HTTPS 地址 |
| Python 依赖缺失 | `python3 -m pip list`；执行 `apt install python3-pip` |
| 权限问题 | `ls -la /opt/srvpulse/` |
| 状态文件异常 | `cat state.json`，必要时删除重置 |
